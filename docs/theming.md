# Theming

This document describes how system-wide theming works on `host-garuda-v7` using
DankMaterialShell (DMS) and matugen.

## Overview

Theming is driven by two tools that work in concert:

- **DMS** (`dms-shell`) — the desktop shell running on top of the niri compositor.
  When a wallpaper is changed (via `Mod+Y`), DMS calls matugen internally and then
  writes updated color files to various locations across the system.
- **matugen** — derives a [Material You](https://m3.material.io/styles/color/overview)
  color palette from a source image using the `scheme-tonal-spot` algorithm (dark mode).
  The palette is then written to output files via templates.

Matugen can also be invoked manually to regenerate all outputs without changing the
wallpaper — see [Manual invocation](#manual-invocation) below.

## Architecture

There are two categories of themed files:

| Category | Description |
|---|---|
| **DMS-managed** | Generated and owned entirely by DMS. Do not edit — changes will be overwritten on the next theme update. These files include their own "do not edit" comment headers. |
| **Matugen-template-managed** | Generated by matugen from user-authored `.tera` templates stored in this repo under `packages/host-garuda-v7/.config/matugen/`. The generated output is committed to the repo so it is always present when the package is stowed. |

## Currently Themed Applications

| Application | File | Managed by |
|---|---|---|
| Niri (compositor) | `packages/host-garuda-v7/.config/niri/dms/colors.kdl` | DMS |
| Neovim | `packages/host-garuda-v7/.config/nvim/colors/base16-matugen.vim` | matugen template |
| WezTerm | `packages/host-garuda-v7/.config/wezterm/colors/dank-theme.toml` | DMS |

### Niri

`dms/colors.kdl` is included into the main niri config via:

```kdl
include "./dms/colors.kdl"
```

It sets focus ring, border, shadow, tab indicator, and insert hint colors. All other
`dms/*.kdl` files are also DMS-managed and should not be edited directly.

### Neovim

Neovim theming uses two files with distinct responsibilities:

**`packages/host-garuda-v7/.config/nvim/colors/base16-matugen.vim`** — the generated
colorscheme file. Contains the full `base16-colorscheme.setup({...})` call with all 16
palette slots, plus all highlight group overrides (`Visual`, `Statusline`, `Function`,
etc.). This file is generated by matugen from the template and committed to the repo.
Do not edit it directly.

**`packages/host-garuda-v7/.config/nvim/lua/plugins/matugen.lua`** — a static lazy.nvim
plugin shim. It sets `colorscheme = "base16-matugen"` scoped to this host, declares
`base16-nvim` with `priority = 1000`, and installs a filesystem watcher that
re-sources the colors file whenever matugen regenerates it.

**`packages/extra-nvim/.config/nvim/lua/plugins/theme.lua`** — sets
`colorscheme = "base16-default-dark"` as a safe fallback for all other hosts where
`base16-matugen.vim` does not exist. On `host-garuda-v7`, `dankcolors.lua` overrides
this to `"base16-matugen"`.

#### LazyVim integration

LazyVim calls `vim.cmd.colorscheme(name)` at startup. This fires a `ColorSchemePre`
autocmd that lazy.nvim intercepts to auto-load the plugin that owns the named
colorscheme. Because `base16-matugen.vim` lives in `~/.config/nvim/colors/`, Neovim
finds it directly — no plugin auto-load is needed, and the file's `setup()` call
applies the palette immediately. The `config` function in `dankcolors.lua` only
installs the watcher; it does not call `setup()` itself.

#### Live reload

`matugen.lua` contains a `libuv` filesystem watcher (`uv.new_fs_event`) pointing
at `~/.config/nvim/colors/base16-matugen.vim`. When matugen regenerates the file
(on wallpaper change via DMS, or manually), Neovim automatically re-sources it and
applies the new palette without requiring a restart.

#### Updating the theme

After running matugen to generate a new palette, commit the updated colors file:

```bash
matugen image /path/to/wallpaper.jpg -t scheme-tonal-spot -m dark
git add packages/host-garuda-v7/.config/nvim/colors/base16-matugen.vim
git commit -m "feat(nvim): update matugen colorscheme"
```

### WezTerm

`dank-theme.toml` defines the full ANSI 16-color palette plus background, foreground,
cursor, and selection colors for the WezTerm terminal emulator. It is referenced by
name in `packages/host-garuda-v7/.config/wezterm/local_config.lua`:

```lua
color_scheme = "dank-theme"
```

## Matugen Templates

Templates live in `packages/extra-matugen/.config/matugen/` and are stowed to
`~/.config/matugen/`. The matugen config at `config.toml` maps each template to its
output path.

| Template | Output |
|---|---|
| `templates/nvim-base16-matugen.tera` | `~/.config/nvim/colors/base16-matugen.vim` |

Templates use the [Tera](https://keats.github.io/tera/) templating language. Color
tokens are accessed via `{{base16.base00.default.hex}}` through
`{{base16.base0f.default.hex}}` for the base16 palette slots, which matugen derives
directly from the wallpaper image.

When DMS changes the wallpaper it invokes matugen, which runs all templates defined
in `config.toml` automatically (requires `runUserMatugenTemplates = true` in DMS
settings).

## Manual Invocation

To regenerate all theme outputs from an existing wallpaper image without going through
the DMS UI:

```bash
matugen image /path/to/wallpaper.jpg -t scheme-tonal-spot -m dark
```

Options:

| Flag | Description |
|---|---|
| `-t scheme-tonal-spot` | Color scheme algorithm (matches DMS default) |
| `-m dark` | Dark mode palette |
| `--dry-run` | Preview what would be generated without writing any files |
| `-v` | Verbose output |
